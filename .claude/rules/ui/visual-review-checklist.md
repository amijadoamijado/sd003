# 視覚的評価チェックリスト

## 位置づけ

> **Work First → 動作確認 → このチェックリストで品質評価。**
> 動いていない画面をスコアリングしても意味がない。

## 適用タイミング

```
コード実装 → push/デプロイ → ブラウザで表示 → スクショ撮影 → ★このチェックリストで評価★
                                                                    ↓
                                                             50/70以上 → 次工程へ
                                                             50/70未満 → 修正してやり直し
                                                             35/70未満 → 修正必須（次工程に進めない）
```

## 7項目スコアリング

各項目 0〜10点で評価する。

### 1. 視覚階層 (0-10)

見出し・本文・補助テキストの区別が明確か。

| スコア | 基準 |
|--------|------|
| 0-3 | 全テキストが同じサイズ・ウェイト、優先度が不明 |
| 4-6 | 見出しと本文の区別はあるが、h2/h3の差が不十分 |
| 7-8 | 3段階以上の明確な階層、コントラストも適切 |
| 9-10 | 一目で情報の優先度がわかる、見出しだけで内容が理解可能 |

### 2. 余白・整列 (0-10)

要素間の間隔が統一されているか、グリッドに揃っているか。

| スコア | 基準 |
|--------|------|
| 0-3 | 余白がバラバラ、要素が左右に揃っていない |
| 4-6 | おおむね揃っているが、一部不統一 |
| 7-8 | 8pxグリッド準拠、余白が一貫 |
| 9-10 | 余白が意図的に設計され、呼吸感がある |

### 3. カラー一貫性 (0-10)

デザイントークンに沿っているか、差し色が制限されているか。

| スコア | 基準 |
|--------|------|
| 0-3 | 色が5色以上散在、統一感なし |
| 4-6 | メインカラーはあるが、一部に逸脱した色 |
| 7-8 | 差し色1色 + セマンティック4色に収まっている |
| 9-10 | デザイントークン完全準拠、色に意図がある |

### 4. タイポグラフィ (0-10)

フォントサイズ・行間・ウェイトが制御されているか。

| スコア | 基準 |
|--------|------|
| 0-3 | デフォルトフォントのまま、サイズがバラバラ |
| 4-6 | フォントは選ばれているが、サイズスケールが不統一 |
| 7-8 | 2種以内のフォント、3-4段階のサイズスケール |
| 9-10 | タイポグラフィだけでサービスの雰囲気が伝わる |

### 5. 状態表現 (0-10)

loading/empty/error各状態のUIが用意されているか。

| スコア | 基準 |
|--------|------|
| 0-3 | 状態UIなし、データがないと空白になる |
| 4-6 | loading状態はある（スピナー等）が、empty/errorが未対応 |
| 7-8 | 4状態すべてにUIあり |
| 9-10 | 各状態で次のアクションが案内される（リトライ、データ追加等） |

### 6. レスポンシブ (0-10)

画面幅変更で崩れないか。

| スコア | 基準 |
|--------|------|
| 0-3 | 画面幅を変えると要素がはみ出す、重なる |
| 4-6 | デスクトップでは問題ないが、768px以下で崩れる |
| 7-8 | 768px以下でも適切に折り返し、操作可能 |
| 9-10 | 全幅で最適化、タッチターゲットも適切 |

### 7. 全体印象 (0-10)

「AI臭い量産テンプレ感」がないか、ブランドが見えるか。

| スコア | 基準 |
|--------|------|
| 0-3 | 典型的なSaaSテンプレート、どの会社のサイトかわからない |
| 4-6 | 無難だが個性なし、カードUIの羅列 |
| 7-8 | サービスの雰囲気が伝わる、カード以外の構成が使われている |
| 9-10 | ブランドが一目でわかる、ロゴを隠しても「らしさ」がある |

## スコアシート

```
| # | 評価項目 | スコア | メモ |
|---|---------|--------|------|
| 1 | 視覚階層 | /10 | |
| 2 | 余白・整列 | /10 | |
| 3 | カラー一貫性 | /10 | |
| 4 | タイポグラフィ | /10 | |
| 5 | 状態表現 | /10 | |
| 6 | レスポンシブ | /10 | |
| 7 | 全体印象 | /10 | |
| **合計** | | **/70** | |
```

## 合格基準

| 合計スコア | 判定 | アクション |
|-----------|------|-----------|
| 50-70 | 合格 | 次工程（レビュー、テスト）に進める |
| 35-49 | 要改善 | 低スコア項目を修正してから次工程へ |
| 0-34 | 修正必須 | 次工程に進めない。デザイン原則を再確認 |

## レビューでの使い方

- REVIEW_REPORTの「UI評価」セクションでこのチェックリストを使用する
- スクショを添付し、各項目のスコアと根拠を記載する
- 合格ライン未達の場合、具体的な改善指示を添えてRequest Changesとする
