# セッション管理プロトコル

SD002フレームワークにおけるセッション管理と継続性の確保。

## セッション記録コマンド

### 保存コマンド
```
/sessionwrite
```

**動作内容:**
1. `.kiro/sessions/session-YYYYMMDD-HHMMSS.md` を履歴として新規作成
2. `.kiro/sessions/session-current.md` を上書き保存
3. 以下の情報を記録：
   - 日時（YYYY-MM-DD HH:MM:SS）
   - プロジェクトパス
   - gitブランチ、最新コミット
   - 完了項目（✅）
   - 進行中項目（🔄）
   - 未解決問題（🔴）
   - 作成/修正ファイル一覧
   - 次回作業予定（P0/P1/P2優先度）
   - 注意事項・メモ

### 読込コマンド
```
/sessionread
```

**動作内容:**
1. `.kiro/sessions/session-current.md` を読み込み
2. 最新の作業状況を把握
3. 要約を表示（タスク数、優先度別リスト等）

## 保存先

| ファイル | 用途 |
|---------|------|
| `session-current.md` | 最新版（常に上書き） |
| `session-YYYYMMDD-HHMMSS.md` | 履歴（タイムスタンプ付き） |

## セッションライフサイクル

### 開始時
1. `/sessionread` で前回状況把握
2. 仕様書の最新状態確認（`.kiro/specs/`）
3. ステアリング確認（`.kiro/steering/`）

### 作業中
- 必要に応じてチェックポイント作成
- 重要な変更前にスナップショット

### 終了時
1. `/sessionwrite` で作業状況記録
2. 次回の作業予定明記
3. git commit（必要に応じて）

## セッション記録テンプレート

```markdown
# セッション継続記録

## セッション情報
- **日時**: YYYY-MM-DD HH:MM:SS
- **プロジェクト**: /path/to/project
- **ブランチ**: feature/xxx
- **最新コミット**: abc1234 - commit message

## 作業進捗サマリー

### ✅ 完了した作業
1. タスク1
2. タスク2

### 🔄 進行中の作業
1. タスク3（50%完了）

### 🔴 未解決の問題
- 問題点と試行した解決策

### 📁 作成/修正したファイル
| ファイル | アクション | 説明 |
|---------|----------|------|
| src/xxx.ts | 変更 | バリデーション追加 |
| tests/xxx.test.ts | 新規 | ユニットテスト |

### 🎯 次回セッションで実施すべき作業

#### 優先度P0（緊急）
- 即座に対応すべき作業

#### 優先度P1（重要）
- 重要だが緊急ではない作業

#### 優先度P2（通常）
- 時間があれば対応する作業

### 💡 注意事項・メモ
- 次回担当者への申し送り事項
```

## AI協調体制との連携

| 記録種別 | 場所 |
|---------|------|
| プロジェクト全体セッション | `.kiro/sessions/` |
| 引き継ぎ事項 | `.kiro/ai-coordination/handoff/handoff-log.json` |

## 禁止事項
- ❌ セッション記録の省略
- ❌ テンプレート形式の無視
- ❌ 次回作業予定の未記載
- ❌ ファイル変更の未記録
