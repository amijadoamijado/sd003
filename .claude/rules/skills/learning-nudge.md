# 学習ナッジ（Learning Nudge）

## 原則

> 同じ修正を2度受けたら、それはルールにすべきサイン。
> ただし自動でルールを作らない。提案し、ユーザーが判断する。

## 背景

Hermes Agent（Nous Research）の「self-improving skill system」を参考にSD003へ導入。
Hermesではユーザー修正がスキルの自動更新をトリガーする。
SD003ではClaude Codeのルールとして、修正パターンを検出し永続化を提案する。

## 修正の検出

セッション中に以下のパターンが発生した時、修正として記憶する:

| パターン | 例 |
|---------|-----|
| 明示的な否定 | 「違う」「そうじゃない」「やめて」「それは不要」 |
| やり直し指示 | 「元に戻して」「やり直し」「前のやり方で」 |
| 方針修正 | 「そうではなく〜」「〜ではなく〜で」「〜の方がいい」 |
| 英語の修正 | "no", "wrong", "not like that", "undo", "revert" |

**注意**: テスト失敗ログや技術的なエラーメッセージは修正ではない。
ユーザーの発話のみが対象。

## sessionwrite時の学習評価

`/sessionwrite` 実行時、セッション記録の備考セクションに以下を含める:

### 修正が0-1回の場合
通常通り。学習提案なし。

### 修正が2回以上の場合
備考セクションに記録:

```markdown
### 備考
- 学習ナッジ: セッション中にN回の修正あり
  - [修正1の要約]
  - [修正2の要約]
  - 提案: `.claude/rules/` にルール追加 / auto-memory更新 / スキル作成
```

完了メッセージの末尾に1行で提案を追記:

```
💡 学習ナッジ: N回の修正パターンを検出。ルール化を検討してください。
```

## 提案の3段階

| 段階 | 条件 | 提案内容 |
|------|------|---------|
| 軽量 | 同一セッション内で2回修正 | auto-memoryに feedback として記録 |
| 中量 | 複数セッションで同種の修正 | `.claude/rules/` にルール追加 |
| 重量 | 複雑なワークフロー修正 | `.claude/skills/` にスキル作成 |

## 実行ルール

- **提案のみ**: ルール/スキルの自動作成は禁止
- **非対話**: 完了メッセージへの追記で完結。AskUserQuestionは使わない
- **非ブロッキング**: sessionwriteの保存フローを中断しない
- **控えめ**: 明確な修正パターンのみ対象。曖昧なケースは無視

## 禁止事項

| 禁止 | 理由 |
|------|------|
| ルール/スキルの自動作成 | ユーザー判断が必要 |
| 毎回の修正で即座に提案 | ノイズになる |
| テストエラーを修正としてカウント | ユーザー発話のみが対象 |
| sessionwriteの保存フローをブロック | autosave互換性維持 |
