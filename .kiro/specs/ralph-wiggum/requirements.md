# Ralph Wiggum 要件定義書

## 基本情報
- **機能名**: Ralph Wiggum - Night Mode Autonomous Execution System
- **バージョン**: 1.1.0
- **ステータス**: 設計中（レビュー反映済）
- **作成日**: 2026-01-04
- **最終更新**: 2026-01-04
- **レビュー**: Architecture/Risk/Integration Agent (3並列レビュー完了)

## 1. 概要

### 1.1 目的
日中のAI協調ワークフローと夜間の自律実行を組み合わせた24時間開発サイクルを実現する。夜間は人間の介入なしに、事前定義されたタスクキューを自律的に処理し、朝には完了報告とともに成果物を提示する。

### 1.2 背景
- 既存の`sd002-loop-*`は日中の短期エラー修正ループ（max-iterations 15-20）
- 夜間の長時間自律実行（max-iterations 60）には別の仕組みが必要
- 自動リカバリー、チェックポイント、週次管理が未整備
- 24時間開発サイクルによる生産性向上が期待される

### 1.3 成功基準
- 夜間キュー（nightly-queue.md）に基づくタスク自動実行
- 7パターンの失敗自動リカバリー（v1.1で追加）
- 週次計画・統計ダッシュボードによる進捗可視化
- 既存sd002-loop-*との二層構造での共存（環境変数分離）
- 朝の確認で完了/ブロック状況が即座に把握可能
- AI協調ワークフローへの自動ハンドオフ（TEST_REPORT生成）

## 2. 機能要件

### REQ-RW-001: 夜間キュー管理
- **概要**: nightly-queue.mdに基づくタスクキュー管理
- **入力**: タスクリスト（優先度、推定反復数、完了マーカー）
- **処理**: 優先度順にタスクを実行
- **出力**: 完了マーカー（`<promise>XXX_DONE</promise>`）
- **優先度**: High

### REQ-RW-002: マルチタスク完了マーカー
- **概要**: タスクごとの完了マーカー管理
- **詳細**:
  - 各タスクに固有の完了マーカー（PC001_DONE等）
  - 全体完了マーカー（NIGHTLY_COMPLETE）
  - ブロック時マーカー（NIGHTLY_BLOCKED）
- **入力**: タスク定義
- **処理**: マーカー検出・状態管理
- **出力**: 完了/ブロック状態
- **優先度**: High

### REQ-RW-003: 7パターン自動リカバリー
- **概要**: 失敗パターンに応じた自動復旧
- **詳細**:
  - **Pattern 1: ビルドエラー** - 型エラー自動修正（最大3回）
  - **Pattern 2: テスト失敗** - 実装修正 > テスト修正 > スキップ
  - **Pattern 3: Lintエラー** - `npm run lint --fix` + 手動修正
  - **Pattern 4: 無限ループ検知** - 適応的検知（編集距離・ファイル重複考慮）
  - **Pattern 5: 外部依存エラー** - 指数バックオフ + サーキットブレーカー
  - **Pattern 6: 想定外エラー** - graceful-exit実行（再開可能）
  - **Pattern 7: リカバリー試行過多** - 3回以上失敗でタスクスキップ + エスカレーション（v1.1追加）
- **入力**: エラー内容、エラー履歴
- **処理**: パターンマッチング、リカバリー実行、エスカレーション判定
- **出力**: 復旧成功/スキップ/終了/エスカレーション
- **優先度**: High

### REQ-RW-004: 自動チェックポイント
- **概要**: 復旧用の状態保存
- **詳細**:
  - 各タスク完了時に自動保存
  - エラー発生時に自動保存
  - 10反復ごとに定期保存
- **入力**: 現在の実行状態
- **処理**: JSONファイルとして保存
- **出力**: `recovery/checkpoints/{timestamp}.json`
- **優先度**: High

### REQ-RW-005: 週次計画管理
- **概要**: 週単位でのタスク割り当て・管理
- **詳細**:
  - 週次計画テンプレート（plan.md）
  - 日別キュー（daily/*.md）
  - バックログ（backlog.md）
- **入力**: ユーザーによる手動計画作成
- **処理**: 計画に基づく日次キュー生成
- **出力**: 各日のnightly-queue.md
- **優先度**: Medium

### REQ-RW-006: 統計ダッシュボード
- **概要**: 実行統計の記録・可視化
- **詳細**:
  - 週次統計（完了タスク数、成功率、消費額）
  - トレンド分析
  - パターン分析（成功/失敗しやすいタスク）
- **入力**: 実行ログ
- **処理**: 統計集計
- **出力**: `metrics/weekly-stats.md`
- **優先度**: Medium

### REQ-RW-007: 品質ゲート統合
- **概要**: 既存品質ゲートとの連携
- **詳細**:
  - npm run build
  - npm test
  - npm run lint
- **入力**: タスク完了状態
- **処理**: 品質ゲート実行
- **出力**: Pass/Fail
- **優先度**: High

### REQ-RW-008: 行き詰まり検知
- **概要**: 進行不可状態の自動検知
- **詳細**:
  - 50反復到達時の強制報告
  - 同一エラー3回連続でスキップ
  - 外部依存エラー継続でモック化
- **入力**: 反復カウント、エラー履歴
- **処理**: 閾値チェック
- **出力**: 継続/スキップ/終了判定
- **優先度**: High

### REQ-RW-009: ログ出力
- **概要**: 実行ログの記録
- **詳細**:
  - 成功ログ: `logs/{date}-result.md`
  - ブロックログ: `logs/{date}-blocked.md`
  - エラーログ: `logs/{date}-errors.md`
- **入力**: 実行結果
- **処理**: ログファイル生成
- **出力**: ログファイル群
- **優先度**: Medium

### REQ-RW-010: Gitコミット自動化
- **概要**: タスク完了時の自動コミット
- **詳細**:
  - タスクごとにコミット（設定可能）
  - コミットメッセージ自動生成
  - 破壊的変更の防止
- **入力**: タスク完了状態
- **処理**: git add + commit
- **出力**: コミット
- **優先度**: Medium

### REQ-RW-011: Adapter-Coreパターン適用（v1.1追加）
- **概要**: SD002のAdapter-Coreパターンに準拠した設計
- **詳細**:
  - QueueAdapter: nightly-queue.md → IQueue変換
  - RecoveryCore: パターンマッチング・実行ロジック
  - CheckpointAdapter: JSON ←→ 内部形式変換
- **入力**: 外部形式データ
- **処理**: Adapterで変換 → Coreで処理
- **出力**: 標準形式データ
- **優先度**: High

### REQ-RW-012: Checkpointマイグレーション（v1.1追加）
- **概要**: Checkpointスキーマのバージョン互換性保証
- **詳細**:
  - バージョン番号付きスキーマ（v1.0, v1.1等）
  - 前方/後方互換マイグレーション関数
  - 破損チェックポイントの検出・復旧
- **入力**: 任意バージョンのCheckpoint
- **処理**: マイグレーション実行
- **出力**: 現行バージョンのCheckpoint
- **優先度**: High

### REQ-RW-013: 同時実行保護（v1.1追加）
- **概要**: 複数インスタンスの同時実行防止
- **詳細**:
  - ファイルベースロック（`.kiro/ralph/.lock`）
  - ロック取得失敗時の警告
  - 古いロックの自動解除（タイムアウト）
- **入力**: 実行開始リクエスト
- **処理**: ロック取得・検証
- **出力**: 実行許可/拒否
- **優先度**: High

### REQ-RW-014: 朝のレビュー自動生成（v1.1追加）
- **概要**: AI協調ワークフローへの自動ハンドオフ
- **詳細**:
  - 夜間完了時にTEST_REPORTを自動生成
  - `.kiro/ai-coordination/workflow/review/` に保存

- **入力**: 夜間実行結果
- **処理**: TEST_REPORTテンプレート適用
- **出力**: TEST_REPORT_{date}.md
- **優先度**: Medium

### REQ-RW-015: Promise Detector（v1.1追加）
- **概要**: 完了マーカーの確実な検出
- **詳細**:
  - 正規表現ベース検出
  - stdout/stderr/ファイル別の検出コンテキスト
  - 信頼度スコア付き検出結果
- **入力**: タスク出力
- **処理**: パターンマッチング
- **出力**: IPromise（マーカー、検出位置、信頼度）
- **優先度**: High

## 3. 非機能要件

### NFR-RW-001: 既存システムとの共存
- 日中: 既存sd002-loop-*（max-iterations 15-20）
- 夜間: Ralph Wiggum（max-iterations 60）
- 相互干渉なし

### NFR-RW-002: 朝の確認容易性
- 完了/ブロック状態が即座に把握可能
- チェックリスト形式での確認項目提示
- 次のアクションが明確

### NFR-RW-003: 安全性
- 破壊的git操作の禁止（force push等）
- 設定ファイル変更の制限
- ロールバック可能な状態維持

### NFR-RW-004: 拡張性
- 新しいリカバリーパターン追加可能
- カスタム完了マーカー対応
- 週次計画テンプレートのカスタマイズ

### NFR-RW-005: 環境変数分離（v1.1追加）
- 夜間モード専用環境変数: `RALPH_*`プレフィックス
  - `RALPH_MAX_ITERATIONS`: 夜間最大反復数（デフォルト60）
  - `RALPH_COMPLETION_PROMISE`: 全体完了マーカー
  - `RALPH_BLOCKED_PROMISE`: ブロック時マーカー
- 日中モード: 既存`SD002_*`を継続使用
- 衝突防止: 両者を明確に分離

### NFR-RW-006: Promise名前空間分離（v1.1追加）
- 夜間モード完了マーカー: `RALPH_`プレフィックス
  - `RALPH_NIGHTLY_COMPLETE`（全完了）
  - `RALPH_NIGHTLY_BLOCKED`（ブロック）
  - `RALPH_{TASK}_DONE`（タスク完了）
- 日中モードとのログ混在時も識別可能

## 4. 禁止事項

1. 夜間モードでのユーザー待機状態（AskUserQuestion後の待機）
2. 破壊的git操作（force push, hard reset）
3. 設定ファイルの無断変更（CLAUDE.md, package.json等）
4. 仕様書なしの実装開始
5. max-iterations超過後の継続

## 5. 実行フロー

```
夜間開始
    │
    ▼
nightly-queue.md読み込み
    │
    ▼
┌────────────────────────────────────────┐
│  タスクループ（優先度順）              │
│  ├── タスク開始                        │
│  ├── 品質ゲート実行                    │
│  ├── エラー時 → リカバリー戦略         │
│  │   ├── 成功 → 次へ                   │
│  │   └── 失敗 → スキップ/終了          │
│  ├── 完了マーカー出力                  │
│  ├── チェックポイント保存              │
│  └── 次のタスクへ                      │
└────────────────────────────────────────┘
    │
    ├── 全完了 → NIGHTLY_COMPLETE
    └── ブロック → NIGHTLY_BLOCKED
    │
    ▼
ログ出力・統計更新
    │
    ▼
夜間終了（朝の確認待ち）
```
