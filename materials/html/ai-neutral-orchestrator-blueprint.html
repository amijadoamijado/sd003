AI中立オーケストレーター

要件定義書 · 2026-07-12

1. 背景

SD003はClaude Code主体で成長したため、役割、hook、Bashパイプラインが製品名に固定されている。Codex、agy、Grokを含む任意AIが司令塔になっても、安全性と完了条件が変わらない実行基盤へ移行する。

2. 現状と前提

現状

Codex向けAGENTS・Skill・Native仕様はあるが、中核実行経路はClaude司令塔、Bash、旧Workflow文書を前提とする。

制約

  • Windows 11 / PowerShellを第一級環境とする
  • 既存差分を自動stage・commitしない
  • 外部AI認証や応答揺らぎとハーネス検証を分離する

3. ゴール

Claude、Codex、agy、Grokの誰が起動しても、同じ状態モデル、同じ安全装置、同じ完了条件でタスクを進められる。

4. アウトプット

  • AI中立契約と状態モデル
  • OS非依存TypeScriptランナー
  • provider/role設定
  • run manifestと成果物manifest
  • Codex司令塔E2E証跡

5. 要件

  • 役割とproviderを設定だけで交換できる
  • provider失敗・成果物不足・禁止操作を必ず失敗にする
  • shellを介さず実行ファイルと引数配列でproviderを起動する
  • 実行状態、stdout、stderr、exit code、成果物をrun ID単位で保存する
  • dirty Git workspaceを既定で拒否する

6. 検証観点

  • Codex司令塔の隔離E2Eが成功する
  • provider異常終了が偽成功にならない
  • clasp deploy / undeployが事前拒否される
  • 期待成果物不足を検出する
  • 既存ユーザー差分を変更しない
  • build・lint・test・sync checkが成功する
Known Unknown: 外部CLIの最新版引数差異はprovider設定へ局所化し、実運用接続時に個別検証する。
