# クライアント別帳票設定管理 - 要件定義書

## 1. サマリー

| 項目 | 内容 |
|------|------|
| **課題** | ファイルアップロードのたびに180種類の帳票から使用する帳票を手動選択している。同じ法人なら毎回ほぼ同じ帳票を使うのに、選択を繰り返す無駄がある |
| **解決** | 法人ごとに帳票の使用パターンをデフォルト設定として保存し、次回アップロード時に自動適用する |
| **効果** | 既存顧客のファイルアップロード時、帳票選択が自動化される。ユーザーは差分のみ修正すればよい |
| **対象ユーザー** | NM001会計データ自動照合システムの利用者 |

---

## 2. 機能一覧

| # | 機能名 | 概要 |
|---|--------|------|
| F-001 | 顧客管理マスター | 法人の会社コード・名前・設定シートURLを一元管理するシート |
| F-002 | 法人専用設定シート | 法人ごとに帳票180種のON/OFFを保存するスプレッドシート |
| F-003 | 会社コード自動抽出 | アップロードされたCSVから会社コードと会社名を自動で取り出す |
| F-004 | デフォルト設定の自動適用 | 既存顧客のファイル検出時、保存済みの帳票設定をUIに反映する |
| F-005 | デフォルト設定の保存 | ユーザーの帳票選択をデフォルト設定として保存する |
| F-006 | 既存システム統合 | 既存のNM001コードベースとの接続ポイント |

---

## 3. 機能詳細

### F-001: 顧客管理マスター

**ストーリー**: システム管理者が法人ごとの設定シートURLを一箇所で管理できる。

#### ルール
- メインスプレッドシートに「顧客管理」シートを作成する
- 列構成: 会社コード（主キー）、会社名、スプレッドシートURL、スプレッドシートID、初回登録日、最終更新日、備考
- 会社コードは一意。重複登録時はエラーを表示して中断する

#### データ

| 入力 | 出力 |
|------|------|
| 会社コード + 会社名 | 顧客管理シートの新規行 |
| 法人専用シート作成完了 | URL・IDの自動登録 |

#### 制約（やらないこと）
- 顧客管理シートの手動編集UIは作らない（スプレッドシート直接編集で対応）
- 顧客の削除機能は初期リリースに含めない

---

### F-002: 法人専用設定シート

**ストーリー**: システムが法人ごとの帳票使用パターンを永続化できる。

#### ルール
- シート名は「[会社コード]_[会社名]_帳票設定」
- 達人シリーズ180帳票をチェックボックス形式で配置する
- 列構成: 帳票ID、帳票名、カテゴリ、デフォルト使用状態（チェックボックス）、最終更新日
- Google Drive内の専用フォルダに保存する
- 作成後、URLとIDを顧客管理マスターに自動登録する

#### データ

| 入力 | 出力 |
|------|------|
| 会社コード + 会社名 | 新規スプレッドシート（180帳票リスト付き） |
| ユーザーのチェック操作 | チェック状態 + 最終更新日の更新 |

#### 制約（やらないこと）
- 帳票リスト自体の編集機能は含めない（帳票マスターは別管理）
- スプレッドシートの権限管理は手動で行う

---

### F-003: 会社コード自動抽出

**ストーリー**: ユーザーがCSVをアップロードするだけで会社コードが自動判定される。

#### ルール
- 抽出の優先順位:
  1. ファイル名から正規表現 `^(\d+)_([^_]+)_` で会社コードと会社名を取得
  2. 失敗した場合、ファイル内容（Shift-JIS CSV 2行目）から1列目「会社コード」・2列目「会社名」を取得
  3. 両方失敗した場合、ユーザーに手動入力ダイアログを表示
- ファイル名抽出が最も高速なため最優先とする

#### データ

| 入力 | 出力 |
|------|------|
| CSVファイル（ファイル名 or 内容） | 会社コード + 会社名 |
| 抽出失敗時のユーザー入力 | 会社コード + 会社名 |

#### 制約（やらないこと）
- CSV以外のファイル形式からの会社コード抽出は対象外
- 会社コードの正当性チェック（外部マスタとの突合）は行わない

---

### F-004: デフォルト設定の自動適用

**ストーリー**: 既存顧客のファイルをアップロードすると、前回の帳票選択が自動で復元される。

#### ルール
- 会社コード抽出後、顧客管理シートで該当レコードを検索する
- 見つかった場合、法人専用シートからチェック状態を読み込む
- UIの帳票選択チェックボックスに設定を自動反映する
- 「デフォルト設定を適用しました」と通知を表示する
- 見つからなかった場合（新規顧客）、何もしない（全チェックボックスが初期状態のまま）

#### データ

| 入力 | 出力 |
|------|------|
| 会社コード | 帳票ごとのON/OFF状態 |

#### 制約（やらないこと）
- デフォルト適用後のユーザーによる変更を妨げない（上書き可能）
- 適用結果の差分表示（前回からの変更点）は初期リリースに含めない

---

### F-005: デフォルト設定の保存

**ストーリー**: ユーザーが帳票を選び終わったら、その選択を次回のデフォルトとして保存できる。

#### ルール
- 帳票選択完了時、「この選択をデフォルト設定として保存しますか？」と確認する
- 「はい」かつ新規顧客 → 法人専用シートを新規作成して保存
- 「はい」かつ既存顧客 → 既存の法人専用シートを更新
- 新規顧客の場合、顧客管理マスターにもレコードを追加する
- 保存完了後、「デフォルト設定を保存しました」と通知する
- 「いいえ」の場合、何もしない

#### データ

| 入力 | 出力 |
|------|------|
| ユーザーの帳票選択状態 + 会社コード | 法人専用シートのチェック状態更新 |
| 新規顧客の場合 | 顧客管理マスターの新規行 + 新規法人専用シート |

#### 制約（やらないこと）
- 保存の自動実行（ユーザー確認なしでの保存）は行わない
- 設定の履歴管理（過去のデフォルト設定の閲覧）は行わない

---

### F-006: 既存システム統合

**ストーリー**: 開発者がNM001の既存コードベースに最小限の変更で統合できる。

#### ルール
- `MainCode.js` の `SHEET_NAMES` 定数に `CLIENT_MANAGEMENT: '顧客管理'` を追加する
- `uploadHandler.js` / `FileUploadHandler.js` に会社コード抽出処理を組み込む
- 帳票リストの取得は既存の `loadTatsujinReportList()` 関数を再利用する
- システム検出ロジック `determineSystemAndReportName` との整合性を保つ
- エラーハンドリングは既存の機構に従う

#### データ

| 入力 | 出力 |
|------|------|
| 既存コードベース | 統合ポイント4箇所の変更 |

#### 制約（やらないこと）
- 既存の処理フローの大幅な変更は行わない
- 既存テストの破壊的変更は行わない

---

## 4. 用語集

| 用語 | 意味 |
|------|------|
| 達人シリーズ | NTTデータの税務・会計ソフト。180種類の帳票がある |
| 会社コード | 法人を一意に識別する数値コード。ファイル名やCSV内に含まれる |
| 法人専用シート | 法人ごとに作成される帳票設定用スプレッドシート |
| 顧客管理マスター | 全法人のメタ情報を集約するシート |
