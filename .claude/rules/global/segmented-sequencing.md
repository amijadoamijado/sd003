# Segmented Sequencing（段取りの分離）

## 原則

> ユーザー確認不要な作業を連続実行 → 最後に1回だけユーザー確認ゲート。
> 省略も早期ブロッキングもしない。両方の問題を段取りで解く。

## 背景

- 「省略」問題: AI が テスト・ユーザー確認をスキップして「完了」と主張する
- 「早期ブロッキング」問題: 毎ステップ確認を挟むと効率が崩壊する

ユーザー明言:
> 「夜間にやらせていたのも時間効率考えてのことだろ。省略するなよ大事な部分を。
>  ユーザー確認必要ならコーディングの後に確認すればいいだろ。確認すらしてないのが問題。
>  段取りよくやれよ。ユーザー確認なしでできる部分からやるとかさ」

答え: **段取りで両方を解く**。非ブロッキングを連続実行、確認を末端に集約。

## 適用

### 実装タスクの段取り（非ブロッキング連続 → 末端で1回確認）

```
[非ブロッキング連続実行ゾーン - ユーザー待ちなし]
  Step 5a: TypeScript strict 通過（tsc --noEmit --strict）
  Step 5b: ESLint 0 件（npm run lint）
  Step 5c: テスト + VTD 通過（npm test && npm run test:validate-data）
  Step 5d: dev server 起動（バックグラウンド）
  Step 5e: ブラウザで主要画面疎通 + スクショ取得（materials/images/）

[ユーザーブロッキングゾーン - 1回だけ]
  Step 6: User Confirmation Gate
    - スクショ + 動作URL + 変更ファイル一覧 をユーザーに提示
    - AskUserQuestion で「画面確認OK？」を問う
    - OK → Step 7（Codex レビュー）
    - NG → 指摘をメモして Step 5a へ戻る
```

**Step 5a-5e の失敗は即停止**（省略禁止）。ただし連続実行中の停止なので、ユーザー待ちは発生しない。
**Step 6 は必須**（省略禁止）。しかし1回にまとまっている（効率維持）。

### タスク分類（3種）

| 種別 | 例 | 実行タイミング |
|------|-----|--------------|
| 完全非ブロッキング | 型修正、リファクタ、依存更新 | いつでも（無人実行可） |
| 途中まで非ブロッキング | UI実装（5a-5eまで） | 5a-5eは連続、Step 6でユーザー確認 |
| 完全ブロッキング | 設計判断、要件変更 | ユーザー会話中 |

## 段取り設計の原則

### 独立タスクは並列化

独立したタスクは並列実行（Explore agent 並列、Codex/Gemini 並列等）:
```
独立タスク A ─┐
独立タスク B ─┼→ まとめてユーザーに確認
独立タスク C ─┘
```

### 依存タスクは直列だが非ブロッキング

依存する複数ステップは、ユーザーを待たずに連続実行:
```
Step 1 → Step 2 → Step 3 → [User Confirm]
```

### ユーザー確認は末端集約

以下は**ユーザー確認不要**（自動進行）:
- 型チェック、Lint、テスト
- dev server 起動、ブラウザでのスクショ取得
- Codex レビュー依頼、レビュー結果の受領

以下は**ユーザー確認必須**（末端でまとめる）:
- スクショを見せる「OK？」
- 設計変更の是非
- デプロイ実行

## 禁止事項

| 禁止 | 理由 |
|------|------|
| Step 5a-5e のいずれかをスキップ | 省略違反 |
| Step 6 の User Confirmation をスキップ | 省略違反 |
| 毎ステップで AskUserQuestion を発火 | 効率違反（早期ブロッキング） |
| 無人実行に UI 変更タスクを混ぜる | ユーザー確認ができない（UI変更は会話中に確認する） |
| 失敗を握り潰して次ステップに進む | 省略違反 |

## 関連原則との整合

- 柱1（Output Primacy）: Step 6 でスクショを見せる = アウトプットをユーザーに届ける
- 柱2（Silent Interior）: 5a-5e は内部作業。ユーザーに見せない
- 柱3（Real Data First）: 5c のテストは実データベース。5e は実データでの画面確認

## 全AIモデル共通

このルールはClaude Code、Codex、Antigravity、Grok全てに適用される。
