1

背景

Claude Codeが生成する要件定義書や仕様書は、これまでmarkdown形式のみだった。markdownはAIにとって読みやすいが、人間が確認する際に「目が滑る」問題がある。特に要件定義書は、ステークホルダーとの合意形成に使われるため、視覚的な階層・図解・軽い操作が求められる。
2

現状と前提

2.1 現状

Blueprint Gate（要件定義スキル）は5段階の対話で要件を引き出し、markdownファイルを出力する。人間が読む際はテキストエディタまたはGitHub上でプレビューする必要がある。

2.2 技術環境と制約

SD003フレームワーク上で動作。Claude CodeがHTMLファイルを直接生成。外部サーバーは不要（file:// プロトコルで閲覧可能）。Google Fonts CDNのみ外部依存。

2.3 スコープ

やること: 自己完結HTML生成スキルの作成、Blueprint GateへのHTML出力モード追加、materials/html/ 保存先の定義
やらないこと: Webアプリ・サーバーの構築、AI向け文書（CLAUDE.md等）のHTML化、カスタム編集UI（ドラッグ・スライダー）

2.4 AIへの注意事項

AI向け文書（SKILL.md、CLAUDE.md、AGENTS.md）は素テキストのまま維持すること。HTML化は人間向け出力のみに適用する。
3

ゴール

要件定義書をブラウザで開いて一覧性高く確認・編集でき、Copy as MarkdownでAIに戻せる自己完結HTML生成スキルをSD003に導入する。
4

アウトプット定義

4.1 成果物

  • html-report スキル（SKILL.md + report-template.html + design-tokens.md）
  • Blueprint Gate SKILL.md の拡張（HTML出力モード追加）
  • file-organization.md の更新（html/ ディレクトリ追加）

4.2 完成条件

  • /blueprint-gate 実行後に materials/html/ にHTMLが生成される
  • HTMLが file:// プロトコルで正しく表示される
  • Copy as Markdown ボタンが機能する
  • Print stylesheet でPDF出力が可能
  • visual-review-checklist で 50/70 以上

4.3 利用者と利用場面

あみおさんがBlueprint Gateで要件定義を行った後、生成されたHTMLをブラウザで確認・編集する。ステークホルダーに共有する際もURLまたはファイルで渡す。
5

要件

5.1 機能要件

  • サイドバー付きナビゲーション（セクション間ジャンプ）
  • contenteditable によるインライン編集
  • Copy as Markdown ボタン（clipboard API）
  • Print stylesheet（PDF出力対応）
  • レスポンシブ（768px以下でサイドバー非表示）

5.2 非機能要件

  • 自己完結（外部依存はGoogle Fonts CDNのみ）
  • file:// プロトコルで動作
  • HTMLファイルサイズ 50KB以下
  • visual-review-checklist 50/70 以上の品質
6

検証観点

  • HTMLが file:// で正しく表示される（フォント読み込み、レイアウト崩れなし）
  • Copy as Markdown ボタンでクリップボードにMarkdownがコピーされる
  • サイドバーのアクティブセクション追跡が動作する
  • contenteditable 領域をクリックして編集できる
  • Print stylesheet でPDF出力が読みやすい
