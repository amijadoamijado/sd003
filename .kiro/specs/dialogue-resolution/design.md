## 2. 詳細設計

### 2.1 Step 1: 認識の言語化

**目的**: AIの「思い込み」を可視化する

**制約**:
- ファイル検索禁止
- コード読み込み禁止
- 現在のコンテキストと学習済み知識のみ使用

**出力フォーマット**:
```markdown
## 該当機能の処理フロー
1. [入力] →
2. [処理A] →
3. [処理B] →
4. [出力]

## 参照しているデータ・状態
- [データ/状態1]
- [データ/状態2]

## 期待される動作
[現在の理解に基づく期待動作]
```

**停止条件**:
```
質問: Step 1の認識説明を確認しました。Step 2に進んでよいですか？
選択肢:
- はい、Step 2に進む
- いいえ、Step 1を修正・補足してほしい
```

### 2.2 Step 2: 仕様との照合

**目的**: AIの認識と仕様書の差異を検出

**入力**:
- Step 1の認識
- 仕様書（.kiro/specs/等）

**処理**:
1. 仕様書の該当セクション読み込み
2. Step 1の説明と比較
3. 差異を処理フローレベルで列挙

**出力フォーマット**:
```markdown
| 項目 | Step1での理解 | 仕様書の記述 | 差異の内容 |
|------|--------------|-------------|-----------|
| [項目1] | [AIの理解] | [仕様書] | [差異] |
| [項目2] | [AIの理解] | [仕様書] | 差異なし |
```

**停止条件**:
```
質問: Step 2の仕様照合を確認しました。Step 3に進んでよいですか？
選択肢:
- はい、Step 3に進む
- いいえ、Step 2を修正・補足してほしい
- 差異について詳しく説明してほしい
```

### 2.3 Step 3: 実装との照合

**目的**: 仕様書と実装の差異を検出

**入力**:
- 仕様書
- 実装コード

**処理**:
1. 実際のコード読み込み
2. 仕様書の期待値と比較
3. 差異を2レベルで列挙:
   - 処理フローレベル
   - 参照データ・状態レベル

**出力フォーマット**:
```markdown
| 箇所 | 仕様の期待 | 実装の実態 | 参照データ・状態の差異 |
|------|-----------|-----------|---------------------|
| [箇所1] | [期待] | [実態] | [差異] |
```

**停止条件**:
```
質問: Step 3の実装照合を確認しました。Step 4に進んでよいですか？
選択肢:
- はい、Step 4に進む
- いいえ、Step 3を修正・補足してほしい
- 特定の差異について詳しく調査してほしい
```

### 2.4 Step 4: 修正計画策定

**目的**: 検出した差異の修正計画を策定

**入力**:
- Step 2, 3の差異報告

**出力**:
1. 修正が必要な箇所（優先順位付き）
2. 各修正の内容（処理フロー観点）
3. 完了条件

**停止条件**:
```
質問: 修正計画を提示しました。実行してよいですか？
選択肢:
- はい、修正を実行する
- いいえ、計画を修正してほしい
- 一部のみ実行する（詳細を指定）
```

### 2.5 差異ログ記録

**保存先**: `docs/troubleshooting/RESOLUTION_LOG.md`

**フォーマット**:
```markdown
## YYYY-MM-DD [機能名・処理名]

### 差異カテゴリ
[ ] 仕様理解の誤り
[ ] 実装の逸脱
[ ] 参照データ・状態の誤り
[ ] 仕様の曖昧さ
[ ] 外部要因・環境（API変更、ライブラリ等）

### 処理フローの差異
期待: [期待されたフロー]
実際: [実際のフロー/理解]

### 参照データ・状態の差異
期待: [期待されたデータ・状態]
実際: [実際に参照していたデータ・状態]

### 解決策
[実施した修正内容]

### 教訓
[再発防止のための気づき]
```

## 3. インターフェース設計

### 3.1 AskUserQuestion パターン

各Stepで使用するAskUserQuestionのパターン：

| Step | 質問 | 選択肢 |
|------|------|--------|
| 1完了 | Step 2に進むか？ | はい / いいえ |
| 2完了 | Step 3に進むか？ | はい / いいえ / 詳細説明 |
| 3完了 | Step 4に進むか？ | はい / いいえ / 詳細調査 |
| 4完了 | 修正実行の承認 | はい / いいえ / 一部のみ |

### 3.2 使用ツール

| ツール | 使用Step | 用途 |
|--------|---------|------|
| AskUserQuestion | 全Step | ユーザー確認 |
| Read | Step 2, 3 | 仕様書・コード読み込み |
| Glob | Step 2, 3 | ファイル検索 |
| Grep | Step 2, 3 | コード検索 |
| Edit/Write | 修正実行 | コード修正 |
