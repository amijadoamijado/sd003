- **MSYS パス変換の罠**: Git Bash で `git show <rev>:.sd/specs/...` を実行すると、`<rev>:` の直後の `.sd/...` を MSYS が Windows パスリスト（`;` 区切り）と誤認し `:` を `;` に変換 → `fatal: ambiguous argument` で 0バイト失敗。`MSYS_NO_PATHCONV=1`（または `MSYS2_ARG_CONV_EXCL='*'`）で回避。`rev:path` 形式を Bash tool で使うとき注意
