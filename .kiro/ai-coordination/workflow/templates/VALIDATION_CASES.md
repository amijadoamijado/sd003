# 検証ケース台帳（Validation Cases Ledger）

## 概要

このファイルはプロジェクトの「業務的正しさ」を検証するためのケース台帳です。
テストコードではなくMarkdownで記述し、非エンジニアでも読み書きできます。

## フォーマット

各検証ケースは以下の構造で記述します:

```markdown
### VC-{NNN}: {検証対象の概要}
- **入力**: {具体的な入力データ}
- **期待出力**: {正しい出力結果}
- **検証根拠**: {なぜこの出力が正しいと言えるか — 法令、業務ルール、設計仕様等}
- **本番データ参照**: {実際のデータとの紐付け}
- **最終検証日**: {YYYY-MM-DD}
- **検証者**: {誰が確認したか}
- **ステータス**: {PASS / FAIL / UNTESTED}
```

## 運用ルール

1. **新機能追加時**: 最低3ケースを台帳に追加する（正常系1、境界値1、異常系1）
2. **バグ修正時**: バグの再現ケースを台帳に追加する（回帰防止）
3. **業務受入時**: クライアント確認の結果を台帳にフィードバックする
4. **棚卸し**: 四半期ごとに全ケースのステータスを再検証する
