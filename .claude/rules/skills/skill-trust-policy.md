# スキル信頼ポリシー

## 概要

SD003では外部スキルエコシステム（skills.sh / `npx skills`）と連携する。
スキルのソースに応じた信頼レベルを定義し、安全なインストールを担保する。

## 信頼レベル定義

| 信頼レベル | ソース | 扱い |
|-----------|--------|------|
| **Trusted** | `anthropics/skills`（Anthropic公式） | 自由にインストール可 |
| **Trusted** | `vercel-labs/skills`, `vercel-labs/agent-skills`（Vercel Labs） | 自由にインストール可 |
| **Local** | `.claude/skills/` 直下（SD003独自スキル） | プロジェクト管理下 |
| **Caution** | その他のリポジトリ | SKILL.md確認後にインストール |

## インストール前チェックリスト（Cautionレベル）

Cautionレベルのスキルをインストールする前に以下を確認:

1. **SKILL.mdの内容確認**: スキルの説明と動作を理解する
2. **scripts/ ディレクトリの確認**: 不明なBashコマンド実行やファイル操作がないか
3. **リポジトリの信頼性**: スター数、メンテナンス状態、作者の信頼性
4. **ユーザーへの確認**: インストール前にユーザーの明示的な承認を得る

## 禁止パターン

以下の特徴を持つスキルはインストール禁止:

| パターン | 理由 |
|---------|------|
| 環境変数やシークレットを外部に送信 | 情報漏洩リスク |
| `rm -rf` 等の破壊的コマンドを含む | データ損失リスク |
| 不明なAPIエンドポイントへの通信 | セキュリティリスク |
| パーミッション変更を含む | システム改変リスク |

## コマンド

| コマンド | 説明 |
|---------|------|
| `/skills:find {query}` | スキル検索 |
| `/skills:add {owner/repo}` | スキルインストール |
| `/skills:list` | インストール済み一覧 |

## 全AIモデル共通

このポリシーはClaude Code、Codex、Gemini CLI、Antigravity全てに適用される。
